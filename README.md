# Исследовательский проект по временным рядам, ВШЭ ИИ 1 курс

### Предмет исследования

В рамках проекта рассмотрена гипотеза: "Для каких типов рядов какая трансформация ряда полезнее?"

*Данные:* просемплированы 200 рядов из датасета M4 Monthly Dataset: https://zenodo.org/records/4656480

*Метрики:* В качестве метрик выбраны MAE и SMAPE. В то время как MAE является классической метрикой, отображающей среднюю абсолютную ошибку, SMAPE показывает относительную ошибку - это удобно в нашей ситуации, где ряды имеют разный масштаб. 

*Преобазования:*
Рассматриваются следующие преобразования:
- log1p
- Box-Cox
- дифференцирование (differencing)

*Цель:* Цель — определить, какие преобразования наиболее эффективны для различных типов временных рядов.

**Гипотеза:**
Предполагается, что:
> различные преобразования по-разному влияют на качество прогнозирования 
> в зависимости от структуры временного ряда (наличия тренда и дисперсии)

## Ход исследований

### Кластеризация
Ряды разбиты на 2 кластера:

- **Кластер 0** — стационарные или со слабым трендом  
- **Кластер 1** — выраженный восходящий тренд  

Метод:
- TimeSeriesKMeans (DTW distance)
- выбор k = 2 (Silhouette)

### Используемые модели

### Бейзлайны:
- Naive
- SeasonalNaive

### Глобальная модель:
- CatBoost (recursive forecasting)

### Метрики
- MAE
- SMAPE

## Основные результаты: 

### Бейзлайны

- log1p и Box-Cox не влияют на качество
- differencing ухудшает качество (особенно для Naive)

Причина:
- бейзлайны зависят от уровня ряда

### CatBoost

#### Кластер 0 (без тренда):
- лучший результат: **log1p**
- Box-Cox близок по качеству
- differencing сильно ухудшает результат

#### Кластер 1 (с трендом):
- лучший результат: **Box-Cox**
- log1p даёт схожее качество
- исходный ряд уже даёт хороший результат
- differencing полностью ломает модель

## Выводы: 

- выбор преобразования зависит от типа временного ряда  
- для рядов без тренда:
  - log1p улучшает качество  
- для трендовых рядов:
  - преобразования не критичны  
  - Box-Cox даёт небольшой выигрыш  

- Дифференцирование не рекомендуется в данной задаче  

- CatBoost значительно превосходит бейзлайны
